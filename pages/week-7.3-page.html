Week 7 - Python Chess Program

WHY?

In my Introdcution to Computer Science class (CS 162), our final project for the course was to build a variation of the classic game of Chess. In this version, all pieces moved the same but the starting position of the pieces is the one shown to the left. However, you are not allowed to make a move that puts the opposing player's king in check, nor your own king in check. The purpose of the game is instead to get your king to row 8 of the board. This was an appropriate challenge for my current skill level with programming in Python, but after finishing the project I felt I could learn a lot from challenging myself to build a full working Chess game with Python. Therefore, my goal this week was to:

Create a playable game of Chess using Python.

HOW?

To start this project, there were a few significant challenges that I could forsee before even beginning. Firstly, I would need to find a way to represent each of the pieces with unique properties, and use some kind of object to represent their positions on the board. The greatest challenge for this project would definitely be to determine if a move is allowed or not. This is especially difficult because of how many checks need to be done for each piece attempting to move. Finally, I would have to account for special cases like En Passant, Castling, promoting a Pawn, Stalemate, and Checkmate.

WHAT?

To start this project I create a class of objects called Pieces that would represent the individual pieces on the board containing information like the piece's name, color, ASCII symbol, and number of moves made. I also defined a unique class for each piece that inherits from the Pieces class and initializes the variables with the correct values. I then defined a ChessVar class that is used to represent the chess game. This class has internal variables for each piece on the board, which is represented by a dictionary containing 8 lists as the keys to correspond to the 8 columns on the chess board. It also has useful variables to track the state of the game (check, unfinished, stalemate, etc.), the turn number, and stats about whether or not someone is attempting a "special" move (en passant, castle, promotion). The class has several methods but the two most important are the make_move() method which calls the verify_move() method. These two work together to ensure that a move is legal and update the board accordingly if it is.
