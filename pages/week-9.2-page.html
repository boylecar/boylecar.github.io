HOW?

In order to continue with this project by building the Program Counter Module, I continued to follow along with the instructions and schematics provided by Ben Eater, an electronics educator. This module will simply act as 4-bit binary counter that will be implemented later to track the current step of the program being executed. To build this module, I only needed two IC chips. The 74LS161 itself is a binary counter which increments by 1 at the rising edge of each clock pulse. The outputs of this chip are put through a tristate buffer (74LS245) to control when it is driven to the bus.

WHAT?

Building this module was relatively simple as I have already used the tristate buffer chip before, and the binary counter chip was simple to use. At a fundamental level this chip uses flip-flops with its outputs tied to its inputs, and its clock signal tied to the output of the previous flip-flop hooked up in series. This will effectively cause the next-most significant bit to turn on when the one below it turns off, which is exactly how you would count in binary. The enable pin for this chip will be used later in the design of the CPU to decide when the program counter should increment or not. The value stored by this counter will represent the "step" of the current program, and increment after a step is completed.
